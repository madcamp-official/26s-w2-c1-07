# LAN 두 컴퓨터 수동 테스트 가이드

같은 공유기(LAN)에 연결된 Windows PC 두 대로 방 생성 → 참가 → 대기방 → 게임 → 대기방 복귀를
검증하는 절차. 방 코드/Relay 없이 **호스트의 사설 IPv4 + UDP 포트 직접 접속** 방식이다.

## 1. 빌드 만들기

1. Unity 에디터에서 `File > Build Settings`(또는 Build Profiles) 열기
2. 씬 목록에 `Assets/Scenes/MainScene.unity` 하나만 체크되어 있는지 확인(현재 기본 상태)
3. 플랫폼 **Windows** 선택 후 Build → 빌드 폴더 생성
4. **빌드 폴더 전체**(exe + `_Data` 폴더 + `UnityPlayer.dll` 등)를 두 컴퓨터에 복사
   - 두 PC 의 빌드는 반드시 **같은 빌드**(같은 버전)여야 한다

## 2. 네트워크 준비

- 두 PC 를 **같은 공유기/같은 LAN** 에 연결(유선/무선 무관)
- 게스트 Wi-Fi, "AP isolation(무선 단말 간 통신 차단)" 이 켜진 네트워크는 서로 접속이 안 됨

## 3. 방 만들기 (컴퓨터 A = 호스트)

1. 게임 실행 → 타이틀에서 닉네임 입력 → **[방 만들기]**
2. 처음 실행 시 Windows 방화벽 허용 창이 뜨면 **"개인 네트워크" 체크 후 허용**
   - 창을 놓쳤다면: Windows 설정 → 방화벽 → "방화벽을 통해 앱 허용" 에서 게임 exe 의
     개인 네트워크를 허용
3. 대기방 상단의 **접속 주소** 확인 (예: `192.168.0.10:7777`)
   - 기본 포트(7777)가 사용 중이면 다른 포트로 자동 대체되고 화면에 그 사실이 표시된다.
     **화면에 표시된 실제 포트**를 참가자에게 알려줄 것
   - 주소가 여러 개 표시되면(VPN/가상 어댑터) 공유기 대역(보통 `192.168.x.x`)을 사용

## 4. 방 참가 (컴퓨터 B)

1. 게임 실행 → 닉네임 입력 → **[방 참가]**
2. 컴퓨터 A 화면에 표시된 **IP 와 포트** 입력 → **[참가]**
   - `127.0.0.1` 이나 `localhost` 는 같은 PC 전용 — 다른 PC 접속에 입력하면 안 됨
3. 접속되면 대기방에 두 명이 표시된다 (방장 표시 = 컴퓨터 A)

## 5. 게임 시작 → 대기방 복귀

1. 두 PC 모두 **[준비하기]** 클릭 → 전원 "준비 완료" 확인
2. 컴퓨터 A(방장)의 **[게임 시작]** 클릭 → 준비(PREP) 페이즈로 진입
3. 3라운드 진행 → 결과 화면 → 결과 시간이 끝나면 **연결이 유지된 채 대기방으로 자동 복귀**
4. 준비 상태가 초기화되어 있는지, 다시 준비 → 게임 시작이 되는지 확인

## 6. 접속이 안 될 때 점검 순서

| 점검 | 방법 |
|---|---|
| 같은 LAN 인가 | 두 PC 의 IPv4 가 같은 대역인지 (`ipconfig`) |
| 게스트 Wi-Fi / AP isolation | 공유기 설정 확인, 일반 SSID 로 재연결 |
| 호스트 방화벽 | 방화벽에서 게임 exe 의 "개인 네트워크" 허용 여부 |
| 포트가 맞는가 | 호스트 **화면에 표시된 실제 포트**(자동 대체됐을 수 있음) |
| 빌드 버전 | 두 PC 의 빌드가 동일한지 |
| 잘못된 주소 | `127.0.0.1`/`localhost` 를 입력하지 않았는지 |
| 핑 확인(선택) | B 에서 `ping 192.168.x.x`(A 의 IP) 응답 여부 |

거절 사유가 있는 실패(방이 가득 참 / 게임이 이미 진행 중)는 참가 화면에 그대로 표시된다.
원인을 알 수 없는 실패는 "서버에 연결할 수 없습니다" 또는 타임아웃으로 표시된다.
