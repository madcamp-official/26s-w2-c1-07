# 클라이밍 전환 명세서 (Only Up 모티브)

> 게임 컨셉 전면 전환(팀 결정, 2026-07-11). 기존 "주제 룰렛 + 수평 코스 레이스" 구조를 폐기하고,
> **플레이어들이 구조물을 설치하며 수직 타워를 오르는 클라이밍 게임**으로 바꾼다.
> Only Up! 에서 모티브만 가져오며 진행 방식은 다르다: 매 라운드 시작 전 준비 페이즈에서
> 구조물을 설치하고, **설치한 구조물이 3라운드 내내 누적**되는 것이 핵심 루프다.
> 이 문서가 신규 컨셉의 단일 기준 명세다. 기존 기획서(docs/기획서.md)의 룰렛·주제·모드 부분은 무효.

## 1. 용어 정의

| 용어 | 정의 |
|---|---|
| 매치 | 준비 → 라운드1 → 준비 → 라운드2 → 준비 → 라운드3 → 결과 로 이루어진 한 판 |
| 라운드 | 플레이(등반) + 하이라이트로 이루어진 단위. 매 라운드 앞에 준비 페이즈가 붙는다 |
| 구조물 | 플레이어가 설치할 수 있는 모든 Object. 보이는 구조물 / 보이지 않는 구조물 두 종류 |
| 보이는 구조물 | 생성된 순간부터 어떤 페이즈에서든 모든 플레이어에게 보이는 구조물 |
| 보이지 않는 구조물 | 상호작용이 없을 때: 준비 페이즈에는 설치자에게만 보이고, 그 외 페이즈에는 아무에게도 안 보임. 플레이어와 상호작용(충돌 등)이 생기면 모든 플레이어에게 일시적으로 보임 |
| 맵 볼륨 | **구조물 생성 범위.** 랜덤 발판 + 플레이어 설치 구조물이 존재할 수 있는 6(가로) x 6(세로) x 50(높이) 사각기둥 공간(개념 경계) |
| 슬라이스 | 맵 볼륨을 높이 기준 x 등분한 6 x 6 x (50/x) 구간. 랜덤 생성의 단위 |
| 지면 | 맨 아래 바닥(y=0) 플랫폼. 맵 볼륨(6x6)보다 넓은 24(가로) x 24(세로) 광장. **플레이어 이동 가능 범위**(경계벽 위치)와 **PREP 페이즈 비행 범위**의 기준이 된다(구조물 생성 범위와는 별개) |

## 2. 매치 흐름 (FSM)

```
LOBBY → [PREP → PLAY → HIGHLIGHT → INTERMISSION] x3 → RESULT → LOBBY
(마지막 라운드는 INTERMISSION 대신 RESULT 로 진행)
```

- 기존과의 차이: **ROULETTE 페이즈 삭제**(주제 랜덤 생성 없음), **PREP 이 라운드 루프 안으로** 이동(매 라운드 전에 1회씩, 총 3회).
- INTERMISSION = 하이라이트 연출 후 다음 라운드 PREP 로 곧장 넘어가면 어색해 두는 라운드 간 완충 대기(`intermissionDuration`, 기본 5초). 구현 중 추가된 페이즈.
- HIGHLIGHT 는 각 라운드마다 유지(라운드 결과 카드).
- 구조물 정리 시점: **RESULT → LOBBY 전환 시에만 전체 despawn.** 라운드 사이(HIGHLIGHT → PREP)에는 절대 지우지 않는다(누적이 핵심).
- 맵 랜덤 생성 시점: LOBBY → 첫 PREP 진입 시 시드 확정(매 매치 새 맵).

## 3. 맵 랜덤 생성

- 맵 볼륨(구조물 생성 범위): 가로(X) `mapWidth`=6, 세로(Z) `mapDepth`=6, 높이(Y) `mapHeight`=50. 랜덤 발판과 플레이어 설치 구조물은 이 좁은 범위 안에만 존재할 수 있다(설치 시 서버가 `InsideVolume`으로 검증).
- 지면: 맵 볼륨보다 넓은 `floorWidth`=24 x `floorDepth`=24 플랫폼(y=0). **플레이어 이동 가능 범위**(사방 투명 경계벽 위치)와 **PREP 페이즈 자유 비행 범위**는 구조물 생성 범위가 아니라 이 지면 크기(24x24)를 따른다 — 플레이어는 구조물이 없는 지면 위 공간도 자유로이 오갈 수 있다.
- 볼륨을 높이 기준으로 `sliceCount`(x)=75 등분한다. 슬라이스 하나 = 6 x 6 x (50/75 ≈ 0.667).
- **각 슬라이스마다** 겹치지 않는 **보이는 구조물을 `structuresPerSlice`(y)=2개** 랜덤 위치에 설치한다(총 150개). 겹침 판정은 AABB 여유 `minStructureSpacing` 기준, 슬라이스당 최대 `maxPlacementAttempts`회 재시도.
- 구조물 종류는 현재 보유 프리팹(벽·원기둥)에서 랜덤, 수평 회전 0/90도 랜덤. 신규 구조물은 추후 추가(프리팹 배열만 늘리면 됨).
- **결정론적 생성:** 호스트가 매치마다 시드 1개를 정해 `NetworkVariable<int> MapSeed` 로 복제하고, 모든 피어가 같은 시드로 동일한 맵을 로컬 생성한다(150개 NetworkObject 스폰 불필요, late-join 자동 대응). 생성 지오메트리는 `Ground` 레이어(기존 MapSurface/카메라 오클루전 호환). 경계벽은 별도 레이어(카메라 오클루전·설치 판정 제외).
- 기존 `Map_FallGuys` 캔디 코스는 폐기(씬에서 제거). 캔디 머티리얼 팔레트는 재사용.

## 4. 플레이어 (이동·점프·체력)

- 이동/시점: 기존 그대로(3인칭 마우스룩 조준 시점, WASD, 커서 잠금).
- **점프력: 스페이스바 점프의 Δy 최댓값 = 1** (`jumpHeight`=1). 슬라이스 높이(0.667)보다 크므로 인접 슬라이스 구조물로 점프 가능.
- **체력: 라운드 시작 시 `roundHp`=10 으로 리셋.** 체력은 서버 전용 상태이며 **어떤 UI 에도 표시하지 않는다**(플레이어에게 비공개 정보).
- **낙하 데미지:** 낙하 높이(공중 최고점 y - 착지점 y)가 `fallDamageMinHeight`=5 **이상일 때만** 적용.
  - 데미지 = `fallDamageBase`(2) + (낙하높이 - 5) x `fallDamagePerMeter`(1.5), 정수 올림.
  - 예: 5 낙하 = 2, 8 낙하 = 7, 10.4 낙하 = 11(즉사). 수치는 전부 시리얼라이즈 필드로 튜닝.
  - 판정 주체: 낙하 궤적은 소유자 클라가 정확히 알므로(소유자 권위 이동) 소유자가 착지 시 `ReportFallServerRpc(fallHeight)` 로 보고하고, **체력 차감·사망 판정은 서버**가 한다.
- **사망(체력 0 이하):** 그 라운드 동안 행동 불가.
  - 서버가 해당 소유자에게 `EliminatedRpc` 전송 → 소유 클라는 입력 잠금, 본체는 사망 지점에 정지(렌더러 off, 콜라이더 off: 좁은 발판에서 시체가 길을 막지 않게).
  - **관전 모드로 전환:** 카메라가 생존 중인 다른 플레이어를 따라다닌다(좌클릭으로 대상 순환). HUD 에 "탈락, 관전 중: P2" 표시.
  - 부활 없음(라운드 종료 시 일괄 복귀).

## 5. 준비 페이즈 (구조물 설치)

- **설치 개수(라운드별 지급, 이월 없음):**

| 시점 | 보이는 구조물 | 보이지 않는 구조물 |
|---|---|---|
| 라운드 1 시작 전 | 3 | 1 |
| 라운드 2 시작 전 | 2 | 2 |
| 라운드 3 시작 전 | 1 | 3 |

  (시리얼라이즈 배열 `visibleAllowance = {3,2,1}`, `invisibleAllowance = {1,2,3}`)
- **설치 방법:** 맵이 높이 50 타워이므로 지상에서는 위쪽에 설치할 수 없다. PREP 동안 **자유 비행 고스트 카메라**로 전환한다: WASD+마우스룩+Space/Ctrl(상승/하강)로 볼륨 안을 날아다니며, 화면 중앙 조준점 레이캐스트 히트점에 좌클릭으로 설치(본체는 바닥 스폰 위치에 잠금).
- 종류 선택: 숫자키 1(벽)/2(원기둥) = 보이는 구조물, 3 = 보이지 않는 구조물. R 키로 90도 회전. 남은 개수는 PREP 패널에 종류별 표시.
- **서버 검증:** PREP 페이즈인지 / 해당 종류 잔여 개수 / 볼륨 경계 안인지 / **플레이어 설치 구조물끼리** 간격(`_minSpacing`, 기본 0.8). 통과 시 서버가 `NetworkObject.Spawn()`(전 클라 복제), 실패는 조용히 거부(프리뷰가 사전 안내). 랜덤 타워 발판과의 겹침은 허용한다: 슬라이스당 2개씩 밀집한 맵이라 물리 겹침 검사를 하면 사실상 설치 불가이고, 타워에 끼워/걸쳐 설치하는 것이 게임적으로도 유효하다(구현 중 실측으로 확정).
- 배치 프리뷰: 조준점 위치에 반투명 고스트 표시, 로컬 예비검증 통과 시 초록/실패 시 빨강.

## 6. 구조물 가시성 규칙 (핵심 표)

| 상황 | 보이는 구조물 | 보이지 않는 구조물 |
|---|---|---|
| PREP, 설치자 본인 | 보임 | **보임(반투명)** |
| PREP, 타인 | 보임 | 안 보임 |
| PLAY/HIGHLIGHT/RESULT, 전원 | 보임 | 안 보임(설치자 포함) |
| 플레이어와 상호작용(충돌) 발생 시 | (항상 보임) | **`revealDuration`(2초) 동안 전원에게 보임** |

- **콜라이더는 항상 전원에게 동일하게 켜져 있다**(물리 공정성 유지, 기존 Ghost 규약 계승). 가시성은 렌더러만 분기.
- 상호작용 감지: 충돌한 플레이어의 소유 클라가 `OnControllerColliderHit` 으로 감지 → `RevealStructureServerRpc(구조물 참조)` → 서버가 구조물의 `NetworkVariable<double> RevealUntil = ServerTime + revealDuration` 기록 → 전 클라가 `ServerTime < RevealUntil` 이면 렌더링. 연속 충돌은 시간 연장.
- 기존 `Obstacle.Ghost`(설치자에게 상시 반투명) 규칙은 이 규칙으로 **대체**된다.

## 7. 점수

- **라운드 점수 = 높이 점수 + 순위 보너스** (호스트 계산, 3라운드 누적):
  - 높이 점수 = (라운드 **종료 시점**의 플레이어 y / `mapHeight`) x `heightScoreMax`(700). 최고점(peak)이 아니라 **종료 시점 높이**라서 마지막까지 안전하게 서 있는 위치가 중요하다. 탈락자는 사망 지점 높이로 계산.
  - 순위 보너스: 종료 시점 높이 내림차순 순위에 `rankBonus = {300, 200, 100}` (4위부터 0). 동률이면 그 높이에 먼저 도달한 쪽이 위.
- 정상(y ≥ mapHeight) 도달 시 높이 점수 만점 고정. 첫 도달자가 나오면 잔여 타이머를 `finishGrace`(15초)로 단축(나머지도 기회는 가짐).
- 라운드 종료 조건: 타이머 만료 / 전원 사망 / 정상 도달 후 유예 종료.
- 매치 승자 = 3라운드 누적 총점 1위. 기존 `NetworkList<RoundResult>` 복제 구조 재사용(Topic 필드는 0 고정).

## 9. 네트워크 계약 (변경 후)

| 종류 | 이름 | 방향 | 설명 |
|---|---|---|---|
| NetworkVariable | `Phase / Round / PhaseEndTime / AliveCount / RoundWinner / MatchWinner` | H→C | 페이즈·타이머·승자 (기존 유지, Topic/Votes 삭제) |
| NetworkVariable | `MapSeed` | H→C | 결정론적 맵 생성 시드(매치당 1회) |
| ServerRpc | `PlaceStructureServerRpc(pos, yawStep, typeByte, invisible)` | C→H | 구조물 설치 요청(서버 검증·스폰) |
| ServerRpc | `ReportFallServerRpc(fallHeight)` | C→H | 착지 낙하高 보고(서버가 체력 차감, 체력은 비공개) |
| ServerRpc | `RevealStructureServerRpc(structRef)` | C→H | 보이지 않는 구조물 충돌 보고 |
| Rpc(Single) | `TeleportPlayerRpc(pos)` | H→소유 클라 | 라운드 시작 바닥 배치(기존 유지) |
| Rpc(Single) | `EliminatedRpc()` | H→소유 클라 | 사망 통보(입력 잠금 + 관전 전환) |
| NetworkVariable | 구조물 `Type / OwnerId / IsInvisible / RevealUntil` | H→C | 구조물 상태(가시성 규칙의 근거) |
| NetworkList | `Results`(라운드별 순위·점수) | H→C | 점수판·하이라이트·결과 (기존 유지) |
| NetworkTransform | 플레이어 위치 | 소유 클라→전체 | 기존 유지 |

체력(HP)은 서버 전용 딕셔너리로만 존재하며 복제하지 않는다(비공개 정보 규칙).

## 11. 결정 사항 (명세 작성자가 정함, 조정 가능)

1. 준비 페이즈 설치는 **자유 비행 카메라**로: 높이 50 타워 특성상 지상 설치만으로는 위쪽 배치가 불가능해서. 본체는 바닥에 잠금.
2. 관전은 **생존자 추적 카메라**(좌클릭 순환)로: "다른 사람의 화면 관전" 요구를 3인칭 추적으로 해석(1인칭 화면 미러링은 네트워크로 불가).
3. 탈락자 시체는 렌더러·콜라이더 off: 좁은 발판 길막 방지.
4. 종료 시점 높이(peak 아님)로 점수 계산, 사망자는 사망 지점 높이.
5. 상호작용 공개 시간 기본 2초, 연속 충돌 시 연장.
6. 타이머 기본값: PREP 60초 / PLAY 180초(밸런싱 대상).
7. 랜덤 구조물은 결정론(시드 복제) 생성: 150개 NetworkObject 스폰 회피, late-join 대응.
