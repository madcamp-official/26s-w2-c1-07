## 1. Day 6 테스트 방법 (에디터 작업 거의 없음)

1. Unity 에디터를 연다(스크립트가 새로 컴파일됨, 콘솔에 에러 0 확인).
2. 그냥 **Play** 를 누른다. HUD 는 `[RuntimeInitializeOnLoadMethod]` 로 **자동 생성**되므로 씬에 아무것도 배치하지 않아도 화면에 뜬다.
3. 좌상단 **Host** 버튼(NetworkBootstrap OnGUI)으로 호스트 시작.
4. Multiplayer Play Mode 가상 플레이어(또는 빌드 2개)로 클라 접속.

확인 포인트:

- 상단 중앙: 페이즈(로비/준비/룰렛/플레이/하이라이트/결과), 라운드 n/3, 남은 시간(초), 모드명
- 우상단 점수판: 플레이어별 색·이름·누적 점수, 1등에 `★`, 내 줄 강조
- 준비: (좌측 하단 PrepClientUI 패널로 장애물 배치·투표) + 상단에 준비 표시
- 룰렛: 모드 이름이 감속하며 돌다가 실제 결정된 주제에 정착
- 플레이: 가운데 하단에 목표 문구(생존 모드는 생존 인원 표시)
- 하이라이트: 라운드 우승자·주제·Top3 카드
- 결과: 전체 화면 최종 순위 + 챔피언
- 각 플레이어 머리 위에 색이 있는 이름표(P1, P2 ...)

### 선택 셋업(원하면)

- **디버그 패널 숨기기**: `MatchManager` 인스펙터의 `Show Debug Gui` 체크 해제(HUD 로 대체됨). NetworkBootstrap 의 Host/Client 버튼은 접속용이라 남겨둔다.
- **플레이어 색 몸체 적용**: `Assets/Prefabs/Player.prefab` 루트(NetworkObject 있는 곳)에 `PlayerTint` 컴포넌트를 추가한다(설정값 없음). 이름표 색과 동일하게 몸체가 칠해져 테스트 때 구분이 쉽다. 정식 캐릭터 모델을 붙이면 그 머티리얼에도 색이 입혀진다(원치 않으면 컴포넌트 제거).
- HUD 를 자동 생성이 아니라 직접 관리하고 싶으면: 빈 GameObject 에 `GameHUD` 를 붙이면 자동 생성 로직이 중복을 막아준다. 자동 생성을 끄려면 `GameHUD.AutoBootstrap` 의 `[RuntimeInitializeOnLoadMethod]` 특성을 지운다.

## 2. Day 7: Windows 빌드

1. `File > Build Profiles`(Unity 6) 또는 `File > Build Settings`.
2. `Scenes In Build` 에 현재 씬(`Assets/test.unity`)이 포함돼 있는지 확인(없으면 Add Open Scenes).
3. 플랫폼 `Windows`(기본) 선택.
4. `Player Settings`:
   - Company Name / Product Name(예: Roulette Party) 설정
   - 기본 해상도/창모드 정도만 확인(그 외 기본값으로 충분)
5. `Build` 클릭 후 출력 폴더 지정 → `.exe` 생성.
6. 멀티 테스트: `.exe` 두 개 실행(하나 Host, 하나 Client, IP `127.0.0.1`).

> Steam 연동(Facepunch Transport + steam_appid.txt 480)은 출시 단계 작업이라 지금은 UnityTransport(IP) 로 충분하다. 트랜스포트 교체 지점은 `NetworkBootstrap.ApplyConnectionData()` 한 곳으로 격리해 두었다.

## 3. 맵 편집 가이드 (폴가이즈 캔디 코스)

절차 지형(TerrainGenerator)은 폐기했다. 현재 맵은 씬의 `Map_FallGuys` 루트 아래 정적 지오메트리다:
출발 광장(Z 61~77) → 징검다리 3개 → 경사로 → 하이 덱(범퍼) → 경사로 → 메인 덱 + 옆길 나선 타워(높이 모드용, 최고 12) → 좁은 다리 3열 → 슬라롬 벽 구간 → 골 광장(GOAL_Z=-70 통과 시 완주). 아래는 핑크 슬라임 바다(시각 전용), 낙사는 `Course.DEATH_Y`(y<-3.7) 판정.

맵 편집 규칙(이것만 지키면 자유롭게 수정 가능):

- **밟는 지오메트리는 전부 `Ground` 레이어(8)** 로 둘 것. 스폰 y 계산·장애물 배치 검증·카메라 오클루전이 전부 `MapSurface`(Ground 레이캐스트) 기반이라, 레이어만 맞으면 어떤 모양이든 코드 수정 없이 동작한다.
- 코스는 `Course.START_Z=68`(출발선) → `GOAL_Z=-70`(결승) 흐름을 유지. 출발선 주변(X 약 -7~9)에는 플레이어 8명이 설 바닥이 있어야 한다.
- 점프 한계: 최대 점프 높이 1.5, 틈은 3 이하가 무난(달리며 점프 시 약 4.5 까지 가능).
- 높이 모드 만점 기준은 y=12(`HEIGHT_MAX`). 최고점을 크게 바꾸면 MatchManager 의 `HEIGHT_MAX` 도 함께 조정.
- 머티리얼은 `Assets/Materials/Map/` 의 캔디 팔레트(M_Candy_*)를 재사용하면 톤이 유지된다.
- ProBuilder 로 구간을 다듬거나 새 구간을 추가해도 좋다(위 규칙만 준수). 수정 후 Ctrl+S 필수(씬 오브젝트라 전 클라 동일, 동기화 불필요).

분위기 업그레이드: 스카이박스(그라데이션), 라이팅 색, 포스트프로세싱(URP Volume, 블룸+비네트)이 가성비가 크다.

## 4. 캐릭터 디자인 착수 가이드

현재 플레이어는 `Assets/Prefabs/Player.prefab`(CharacterController + PlayerController + ClientNetworkTransform)이다. 모델은 자유롭게 교체 가능하다.

붙이는 순서:

1. 저폴리 캐릭터 모델(fbx/직접 제작)을 Player 프리팹의 **자식**으로 넣는다(루트의 CharacterController/NetworkObject 는 유지).
2. 모델 크기를 CharacterController 캡슐(반경/높이)에 맞춘다. 발이 원점(y=0) 근처에 오도록 오프셋.
3. 색 구분이 필요하면 루트에 `PlayerTint` 를 두면 모델 머티리얼에 플레이어 색이 입혀진다.
4. 애니메이션: `Animator` 를 모델에 추가하고, `PlayerController` 의 이동 입력/속도(이미 계산됨)로 Idle/Run/Jump 파라미터를 구동한다. (이동 로직은 소유자 로컬에서 돌고 위치는 ClientNetworkTransform 로 복제되므로, 애니메이션은 각 클라가 로컬 속도로 재생하면 충분하다.)
5. 이모트(춤 등 숏폼용): 키 입력(신규 Input System, `Keyboard.current`)으로 Animator 트리거를 당기고, 필요 시 간단한 RPC 로 다른 클라에 브로드캐스트한다.

> 우선순위: 걷기/점프가 자연스러운 1종 캐릭터 → 색 구분 → 이모트 1~2개. 캐릭터가 준비되면 낙사 물보라 등 파티클과 사운드를 얹으면 '웃긴 장면'(숏폼) 효과가 커진다.

## 5. 알아둘 점 / 한계

- HUD 한글은 OS 폰트(Malgun Gothic 등)를 동적으로 로드해 렌더한다. Windows 빌드에서 정상 동작(윈도우 기본 탑재). 비한글 OS 로 배포할 경우 한글 폰트 동봉이 필요할 수 있다.
- 하이라이트는 현재 2D 요약 카드(우승자·주제·Top3)다. 3D 킬캠 리플레이는 확장(선택) 항목으로 남겨둔다.
- 점수판 누적 점수는 라운드 종료 시 확정되는 `Results` 기반이라, 진행 중 라운드의 실시간 점수는 라운드가 끝나야 반영된다(설계 의도).
