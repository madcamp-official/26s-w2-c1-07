# 조준(마우스룩) 시점 기능 명세서

> **[아카이브 — 구현 완료]** 마우스룩 조준 시점은 이 명세대로 구현 완료됐다(전 페이즈 커서 잠금,
> 3인칭 궤도, `AimRay`/`AimPoint` 훅). 이후 개편으로 달라진 부분: 어깨 오프셋 기본값은
> 센터 뷰(`aimShoulder=0`), 3.1절 PREP 키 매핑(장애물 1/2/3 + 투표 4/5/6)은 컨셉 전환으로
> 폐기되어 **현행은 슬롯 바 [Alt]/[1]/[2] + 회전 [R]/[T]/[G] + F1 도움말**(README 6절 참조).
> 기능 명세 이력으로 보존한다.

## 1. 개요
- **현재**: 카메라가 캐릭터 이동 방향을 자동으로 뒤에서 추적하고, 캐릭터는 이동 방향으로 회전한다. 마우스는 (준비 페이즈 장애물 배치 외에는) 시점에 관여하지 않는다.
- **목표**: 배틀그라운드 3인칭처럼 **마우스로 시점(=조준)을 제어**한다. 마우스 이동으로 카메라를 궤도 회전, 화면 중앙에 조준점, 이동은 카메라 기준 스트레이프, 플레이 중 커서 잠금.
- **이번 범위**: 조준·시점 제어까지만. **발사(무기/피격)는 포함하지 않는다.** 다만 나중에 발사를 붙일 수 있도록 조준 지점(월드 타깃) 훅(`AimRay`, `AimPoint`, `IsAiming`)만 공개한다.
- **시점**: 3인칭(오버숄더 궤도). 캐릭터·낙사 연출이 계속 보이도록 1인칭은 쓰지 않는다.

## 2. 활성 조건
조준모드는 **모든 페이즈에서 켠다**(`useAimView=true`). 후방추적(비조준) 시점은 코드로 보존하며 `useAimView=false` 로 전환 가능(회귀 대비/취향).

| 상황 | 카메라 | 커서 |
|---|---|---|
| 전 페이즈(로비/준비/룰렛/플레이/하이라이트/결과) | 마우스룩 궤도(조준) | 잠금(숨김) |
| Esc 메뉴 모드 | 그대로(회전 정지) | 자유(에디터 클릭·디버그) |

> 준비(PREP)는 자유 커서로 버튼을 클릭하던 방식이었으나, 전 페이즈가 커서 잠금이 되므로 잠긴 커서에 맞게 재설계했다(3.1 참조). 장애물 배치는 화면 중앙 조준점 기준.

## 3. 입력 (신규 Input System)
| 입력 | 동작 |
|---|---|
| 마우스 이동 delta | yaw(수평)·pitch(수직) 회전. pitch 는 [minPitch, maxPitch]로 클램프. invertY 옵션. |
| W / S | 카메라 전방 / 후방 이동 |
| A / D | 좌 / 우 스트레이프 |
| Space | 점프(기존 유지) |
| 좌클릭 | 조준모드에서 커서 재잠금(창 밖에 나갔다 돌아올 때) |
| Esc | 커서 잠금 해제 토글(에디터 클릭·디버그용) |

마우스 delta 는 프레임 값 그대로 감도만 곱한다(Time.deltaTime 곱하지 않음).

### 3.1 준비(PREP) 조작 (잠긴 커서 기준)
커서가 중앙에 잠겨 있으므로 준비 조작은 키보드 + 조준점으로 한다.

| 입력 | 동작 |
|---|---|
| 1 / 2 / 3 | 장애물 종류 선택: 벽 / 원기둥 / 투명벽 |
| 좌클릭 | 화면 중앙 조준점 위치에 장애물 배치(커서 잠금 상태에서만) |
| 4 / 5 / 6 | 주제 투표: 달리기 / 높이 / 생존 |

좌측 하단 PREP 패널은 현재 선택·남은 개수·표수·키 안내를 보여주는 정보 패널이다(클릭 버튼 아님).

## 4. 카메라 (3인칭 오버숄더 궤도)
- 피벗 = 플레이어 머리 높이(`aimHeight`) + 오른쪽 어깨 오프셋(`aimShoulder`). 캐릭터를 화면 중앙에서 살짝 비켜 배치한다.
- 회전 = `Quaternion.Euler(pitch, yaw, 0)`. 카메라는 피벗 뒤 `aimDistance` 거리에 위치.
- 카메라 전방 = 조준 방향. 화면 중앙 조준점이 그 방향을 가리킨다.
- 지형/장애물 오클루전(SphereCast, `camCollisionMask`)과 지형 높이 클램프는 기존 로직을 공유한다(벽 뚫음·지면 관통 방지).

## 5. 캐릭터 회전
- **조준모드**: 몸이 카메라 yaw 방향을 향한다(스트레이프 이동). 화면 중앙이 곧 정면.
- **비조준모드**: 기존처럼 이동 방향으로 몸이 회전한다.

## 6. 조준점 / 타깃 (발사 확장 훅)
- 화면 중앙 조준점 UI(GameHUD 에 추가, PLAY 중 표시).
- `PlayerController` 공개 프로퍼티(소유자 로컬에서 매 프레임 갱신):
  - `bool IsAiming` : 현재 조준모드 여부
  - `Ray AimRay` : 카메라 중앙에서 나가는 조준 레이
  - `Vector3 AimPoint` : 레이 히트 지점(없으면 원거리 점)
- 향후 발사 기능은 이 값을 그대로 사용한다(이번엔 사용처 없음).

## 7. 커서
- 조준모드: `CursorLockMode.Locked` + 숨김.
- 비조준모드: `CursorLockMode.None` + 표시.
- Esc 로 수동 해제(테스트/에디터 클릭). 조준모드에서 좌클릭하면 재잠금. 디스폰 시 커서 복구.

## 8. 파라미터 (인스펙터, PlayerController)
- 마우스룩: `mouseSensitivity`, `invertY`, `aimMinPitch`(-35), `aimMaxPitch`(70), `aimDistance`(4.5), `aimShoulder`(0.7), `aimHeight`(1.6), `faceCourseOnAimStart`(true), `aimRayDistance`(500).
- 후방추적(기존 유지): `camDistance`, `camHeight`, `camPitch`, `camSphereRadius`, `camMinDistance`, `camCollisionSkin`, `camGroundClearance`, `camCollisionMask`.

## 9. 네트워크
- 카메라·커서·조준점·마우스룩은 **전부 소유자 로컬** 처리다(복제 불필요). 몸 회전만 기존처럼 `ClientNetworkTransform` 으로 복제된다. 권위 이동 계약(소유자 이동, 호스트는 READ)은 그대로다.

## 10. 비목표 (이번 범위 밖, 후속)
- 발사/무기/투사체/피격 판정.
- FPS 1인칭 및 TPS↔FPS 토글.
- 정조준(ADS) 정밀 줌, 반동, 히트마커.

## 11. 수용 기준
- PLAY 진입 시 커서가 잠기고 마우스로 카메라가 부드럽게 회전한다(상하 클램프 동작).
- WASD 가 카메라 기준으로 동작하고 몸이 카메라 방향을 향한다.
- 화면 중앙 조준점이 보이고, 카메라가 벽/언덕을 뚫지 않는다.
- 준비 페이즈로 돌아가면 커서가 풀리고 장애물 배치가 정상 동작한다(회귀 없음).
- Esc 로 커서를 풀어 에디터를 클릭할 수 있고, 게임 클릭으로 다시 잠긴다.
